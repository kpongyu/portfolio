MHelper—
Find Voluntary work around!
  • My Role: User Experience Designer
  • Duration: 3 Months
  • Client: Washtenaw County, Michigan
  • Tools & Methods: Sketch 3, Photoshop, Hype3

MHelper connects volunteer work in a timely manner

Current voluntary work face the big problem of not in real time, when people in community (especially elderly) need help, it is hard for them to find immediate voluntary help; while when volunteers are ready to help, people may don't have the need any more. MHelper is an application to solve the dilemma, it tries to link the resident, the community, and volunteers together. Based on our user research, the needs of residents tend to be immediate, short-term, and location based. So we focus on three types of voluntary work in the application, the housework, shopping, and ride. People publish needs on the platform to link each other.

Design Iterations and Interface Gigs

The initial design approach contains the following 3 major functions:

  • Geo-based task finder
  • Calendar based task managing system
  • Record system for feedback

The feedback from initial user testing reveal the following concerns:

  • The filter should be integrated into the map, not separate.
  • Calendar should be weekly focused instead of monthly.

Based on the feedback, I make iterative re-design of the application, especially focusing on the following aspects:

  • Redesign the filter of the map, put the time filter onto the map
  • Change the calendar interface to a list style, which only contains the most urgent tasks
  • By using bottom sheet and card concepts, reduce the total number of screens of the application.

Further user testing is made for this version of design, despite the minor changes most part are accepted by user, the community center is satisfied and the application will be developed by engineer students.

  • Filter the task directly on screen

    One big concern from many users is it is really hard for them to keep track of the tasks, since the control of time intervial is seperated from the map, the shift from filter panel and the map causes change blindness. Since the control of time period is such a frequent function, I put it directly on the same panel with the map. Which makes it much easier for user to select.

  • Use bottom sheet to show
    task details

    Another design decision is instead of bringing user to a brandnew task page, I use the "bottom sheet" to display the detail of the task once user choose it. Which avoid the trouble for user to go back to maps if they want to see other tasks.

  • Emphasis the most recent tasks

    One thing I learnt from user testing is people are more focus on the events that would happen in recent week, instead of the ones will happen in a month. So instead of making a calendar, I use a list to display just the most urgent voluntary tasks.
